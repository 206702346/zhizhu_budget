# Zhizhu 小账本——个人记账微信小程序 程序说明文档

---

## 一、项目概述

**小账本**是一款基于微信小程序 + 微信云开发构建的全功能个人记账应用。用户可快速记录日常收入与支出，按月份查看账单明细，通过可视化图表分析消费结构，设置月度预算控制开销，并导出账单数据图片保存到本地相册。

| 属性 | 说明 |
|---|---|
| 应用类型 | 微信小程序（原生框架） |
| AppID | `wx3d2ab631b16ed4e3` |
| 基础库版本 | ≥ 2.20.1 |
| 后端服务 | 微信云开发 (CloudBase) |
| 云环境 ID | `cloud1-d7g2k7k3b1809fd31` |
| 编程语言 | JavaScript (ES6+) |
| 包依赖 | 无 npm 依赖，零第三方库 |
| 适用平台 | iOS / Android 微信客户端 |

### 功能清单

| 功能模块 | 说明 |
|---|---|
| 🔐 用户登录 | 微信一键登录，自动获取 openid |
| 📝 新增账单 | 选择收支类型，输入金额、分类、日期、备注 |
| ✏️ 编辑账单 | 点击账单进入编辑弹窗，修改任意字段 |
| 🗑️ 删除账单 | 弹窗确认后安全删除 |
| 🔍 搜索筛选 | 按关键词（备注/分类）+ 金额范围筛选 |
| 📅 月份切换 | 按月快速查看历史账单 |
| 📊 数据统计 | 环形饼图（支出占比）+ 柱状图（6月趋势）+ 分类排行 |
| 💰 预算管理 | 设置月度预算、进度条展示、超支红色警告 |
| 📂 分类管理 | 收入/支出分类分别自定义，支持增删和一键恢复默认 |

## 三、数据库设计

本系统使用微信云数据库（JSON 文档型），共设计 3 个数据集合。

### 3.1 bills 账单表

存储每一笔收支记录。

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `_id` | string | 自动 | 文档唯一标识 |
| `_openid` | string | 自动 | 用户 openid，云函数注入，实现多用户隔离 |
| `type` | string | 是 | 收支类型：`"income"`（收入）或 `"expense"`（支出） |
| `amount` | number | 是 | 金额，存储为数字类型 |
| `category` | string | 是 | 分类名称，如"餐饮""工资"等 |
| `date` | string | 是 | 日期，格式 `YYYY-MM-DD`，如 `2026-06-14` |
| `note` | string | 否 | 备注信息 |
| `createTime` | Date | 自动 | 创建时间，`db.serverDate()` |
| `updateTime` | Date | 自动 | 更新时间，`db.serverDate()` |

### 3.2 budgets 预算表

存储每个用户每月的预算设置。

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `_id` | string | 自动 | 文档唯一标识 |
| `_openid` | string | 自动 | 用户 openid |
| `month` | string | 是 | 月份，格式 `YYYY-MM`，如 `2026-06` |
| `amount` | number | 是 | 预算金额 |
| `createTime` | Date | 自动 | 创建时间 |
| `updateTime` | Date | 自动 | 更新时间 |

### 3.3 categories 分类表

存储用户自定义的收支分类。

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `_id` | string | 自动 | 文档唯一标识 |
| `_openid` | string | 自动 | 用户 openid |
| `name` | string | 是 | 分类名称 |
| `type` | string | 是 | 分类类型：`"income"` 或 `"expense"` |
| `sort` | number | 是 | 排序序号 |

### 3.4 数据库安全

所有集合权限设置为「仅创建者可读写」。云函数通过 `cloud.getWXContext().OPENID` 获取当前用户身份，所有数据库操作都附加 `_openid` 条件进行用户数据隔离，确保用户只能操作自己的数据。

**核心功能**：

**(1) 月份切换**

使用微信 `picker mode="date" fields="month"` 组件实现月份选择器。切换后重新查询云函数 `getBills({month})`，同时更新统计汇总。

**(2) 搜索筛选**

搜索栏可折叠展开。支持两种筛选方式：
- **关键词搜索**：云函数使用 `db.RegExp` 对 `note`（备注）和 `category`（分类）字段做模糊匹配
- **金额范围筛选**：在客户端对云函数返回的结果进行金额范围过滤

搜索后即时显示结果数量，清除搜索条件恢复到当月全部账单。

**(3) 编辑账单**

点击"编辑"按钮弹出底部弹窗（bottom sheet），包含与原新增页面一致的表单项。修改后调用 `updateBill` 云函数，通过 `_id + _openid` 双重条件校验权限后更新。

**(4) 删除账单**

点击"删除"按钮，使用 `wx.showModal` 弹出确认对话框，用户确认后调用 `deleteBill` 云函数删除。

**(5) 数据导出**

点击"导出"按钮，调用 `export.js` 模块：
1. 获取当月全部账单数据
2. 使用离屏 Canvas（`type="2d"`，隐藏在屏幕外）渲染精美的月度统计报告图片
3. 通过 `wx.canvasToTempFilePath` 将 Canvas 导出为 PNG
4. 调用 `wx.saveImageToPhotosAlbum` 保存到用户手机相册

导出的报告图片包含：渐变深色头部（大数字结余）、收入/支出双栏、三列摘要卡片、支出分类排行、账单明细列表、应用页脚。

### 5.2 关键实现细节

**用户权限控制**：所有涉及用户数据的操作，云函数内部均通过 `cloud.getWXContext().OPENID` 获取当前用户的 openid，在数据库查询/更新/删除时附加 `_openid` 条件。例如：

```js
// 更新账单时双重验证 _id 和 _openid
const result = await db.collection('bills')
  .where({ _id, _openid: openid })
  .update({ data: updateData })
```

**月份筛选**：使用云数据库正则表达式匹配 `date` 字段前缀：

```js
conditions.date = db.RegExp({
  regexp: `^${month}`,   // '^2026-06'
  options: ''
})
```

**预算 upsert**：`setBudget` 采用"先查后写"策略——若当月已有预算记录则更新，否则新增。

**分类保存**：采用"先删除后写入"的全量替换策略，按 type 分别管理，保证支出和收入的分类不会互相覆盖。

## 六、前端工具模块

### 6.1 cloud.js —— 云函数调用封装

统一封装 `wx.cloud.callFunction`，将微信回调式 API 转换为 Promise 风格，便于页面使用 `async/await`。所有接口自动提取 `res.result.data`，当 `success: false` 时自动 reject。

### 6.2 chart.js —— Canvas 图表绘制

使用 Canvas 2D API 自绘图表，零第三方依赖。

| 函数 | 功能 | 技术要点 |
|---|---|---|
| `drawPieChart(ctx, data, w, h)` | 环形饼图 | 外径/内径（58%）绘制扇形，12 色配色方案，中心显示分类数，右侧自适应图例 |
| `drawBarChart(ctx, data, w, h)` | 柱状趋势图 | 收支双柱并列，圆角柱顶，Y 轴 4 段网格 + 自适应单位（k/w），月份标签 |

图表需用 `wx.createSelectorQuery()` 获取 Canvas 节点，并适配设备像素比 (DPR)：

```js
const dpr = wx.getSystemInfoSync().pixelRatio
canvas.width = res[0].width * dpr    // 物理像素
canvas.height = res[0].height * dpr
ctx.scale(dpr, dpr)                   // 缩放上下文
```

饼图配色方案：`#ef4444 #f97316 #f59e0b #10b981 #3b82f6 #8b5cf6 #ec4899 #06b6d4 ...`（12 色循环）

### 6.3 export.js —— 数据导出为图片

使用离屏 Canvas 渲染精美的统计报告图片。

| 函数 | 功能 |
|---|---|
| `exportMonthReport(data)` | 渲染 750px 宽渐变卡片：头部（深色渐变 + 大数字结余）、收支双栏、摘要卡片、分类排行、账单明细（最多8条）、页脚 |
| `saveToAlbum(tempFilePath)` | 检查 `writePhotosAlbum` 权限 → `wx.saveImageToPhotosAlbum`，无权限时引导用户开启 |


图 7.1 用户登录流程

### 7.2 数据隔离机制

本系统基于微信 openid 实现多用户数据隔离：

1. 用户在客户端无法指定 `_openid` 字段
2. 所有数据写入操作在云函数中执行，由服务端通过 `WXContext.OPENID` 获取真实用户身份
3. 所有查询、更新、删除操作均附加 `_openid: openid` 条件
4. 不同用户的数据在物理层面存储于同一集合，逻辑层面完全隔离

## 十一、部署说明

### 环境要求
- 微信开发者工具（最新版）
- 已注册微信小程序，开通云开发

### 部署步骤

1. 将项目文件夹导入微信开发者工具
2. 修改 `miniprogram/envList.js` 中的 `envId` 或在 `app.js` 中直接修改
3. 右键 `cloudfunctions/quickstartFunctions` → 「上传并部署：云端安装依赖」
4. 在云开发控制台创建 3 个数据库集合：`bills`、`budgets`、`categories`，权限设为「仅创建者可读写」
5. 点击「编译」运行
